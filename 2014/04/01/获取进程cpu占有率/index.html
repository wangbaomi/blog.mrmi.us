Windows在使用期间，有时候会变得很慢很慢，所以要记录一下是哪些进程大量占有了cpu时间。

  • 首先将系统中所有的进程枚举出来
  • 然后一个个地计算每个进程的cpu占用时间

使用NtQuerySystemInformation

枚举所有进程

MS的网站上给出了几种方法:

  1. 使用 ToolHelp32 库

    使用 ToolHelp32 函数的第一步是创建系统信息的"快照"。可以使用 CreateToolhelp32Snapshot() 函数完成这一步。此函数允许选择在快照中存储的信息类型。如果您需要进程信息，则一定要包括 TH32CS_SNAPPROCESS 标志。CreateToolhelp32Snapshot() 函数返回一个句柄，在使用完该句柄后，必须将其传递到 CloseHandle()。

  2. 使用 PSAPI 库

    调用 EnumProcesses().

第一种方式不知道MS以后还支不支持，没有试过；第二种方式在枚举进程时枚举出来的进程不全，没有深入研究；后来找到了使用NtQuerySystemInformation来枚举所有进程并计算cpu占用率的现成代码，便直接拿来用了。

代码放在：https://github.com/wangbaomi/CpuInfo

测试时发现，在win7 32位和win2012环境下都能正常使用，但在win7 64位环境下枚举到的进程数却是0...

一些笔记

  1. 指针不能随便强转
  2. 程序需要提权EnableDebugPrivilege()
  3. 获取本地时间，使用GetLocalTime()
